# Jargon Translation Guide

Quick-reference table for translating technical terms into plain stakeholder language.
Use during the DRAFT phase when translating engineer-authored context into comment copy.

---

## Common Translations

| Avoid (technical) | Say instead (plain language) |
|-------------------|------------------------------|
| "Refactored the codebase" | "Reorganized the code to make future changes faster" |
| "Fixed a null pointer exception" | "Fixed an error that caused the page to crash in certain situations" |
| "Deployed to staging" | "Uploaded the update to our test environment" |
| "API integration" | "Connection between [System A] and [System B]" |
| "The build is broken" | "A recent change caused an issue that's blocking testing — being investigated now" |
| "Technical debt" | "An area that needs cleanup to prevent future problems" |
| "Sprint capacity" | "Available time this week" |
| "Merge conflict" | "Two overlapping changes that need to be reconciled before proceeding" |
| "CI/CD pipeline" | "Our automated build and publish process" |
| "Regression" | "A previously working feature that was accidentally broken" |
| "Code review" | "A team member reviewed the work before it was finalized" |
| "Hot fix" | "An urgent fix applied directly to the live system" |
| "Dependency" | "Another piece of work that must be completed first" |
| "Environment variable" | "A configuration setting specific to the server or environment" |
| "Schema migration" | "A database structure update" |
| "Unit tests" | "Automated checks that verify individual features work correctly" |
| "Load testing" | "Testing how the system performs under heavy use" |
| "Rate limiting" | "A limit on how many requests can be made in a given time period" |
| "Cache invalidation" | "Clearing saved data so the system fetches the latest version" |
| "Feature flag" | "A switch that controls whether a feature is visible to users" |

## Phrases to Avoid

| Avoid | Why | Replace with |
|-------|-----|-------------|
| "Hopefully by Friday" | Erodes confidence; implies uncertainty without stating it | "On track for Friday" or "Expected Friday — depends on [condition]" |
| "Should work now" | Implies the fix is untested | "Fixed and verified" or "Fixed — testing underway" |
| "We're in the home stretch" | Idiom; unclear to international readers | "We're nearly finished" |
| "The ball is in your court" | Idiom; unclear; no explicit ask | "We need your input to proceed" |
| "Move the needle" | Idiom; vague | "Make measurable progress on [specific item]" |
| "Please be advised that" | Corporate stiffness | Remove; begin with the actual statement |
| "At this point in time" | Filler | "Currently" or just state the fact |
| "Going forward" | Filler | Remove; use present or future tense directly |
| "It would appear that" | Hedging | "It appears that" or state the fact directly |
| "Leverage" (as a verb) | Jargon | "Use" |
| "Utilize" | Over-formal | "Use" |
| "Facilitate" | Vague | Describe the specific action |
