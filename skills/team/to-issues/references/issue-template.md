# GitHub Issue Template

Format for each issue generated by the `to-issues` skill.

---

```markdown
## [Issue title — imperative verb phrase: "Add X", "Implement Y", "Fix Z"]

**Type:** feature | bug | chore | docs | test
**Estimate:** [hours or S / M / L]
**Depends on:** [#issue-number or "none"]
**Blocks:** [#issue-number or "none"]

### Description

[What needs to be built and why. 2–4 sentences. Include enough context for a new
engineer to understand the scope without reading the full PRD.]

### Acceptance Criteria

- [ ] [specific, binary criterion — see acceptance-criteria-guide.md]
- [ ] [criterion]

### Notes

[Implementation hints, constraints, or edge cases to keep in mind.
Not prescriptive — engineering chooses the how.]

### Labels

`type:feature` | `type:bug` | `type:chore` | `size:S` | `size:M` | `size:L`
```
